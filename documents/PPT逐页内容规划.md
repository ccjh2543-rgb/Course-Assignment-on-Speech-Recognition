**右栏：Faster-Whisper-small**

```
▪ 参数量：244M
```

**底部加速比卡片**：（大字突出显示）

**解读要点**：
```
▪ 为什么Paraformer更快更准？

  自回归(Whisper)逐token输出 → O(T×L)
  Paraformer训练数据含6万小时中文，Whisper为多语言

▪ 为什么做这个对比？
  ✓ 验证Conformer方案在中文场景的选型合理性
  ✓ 揭示"专用中文模型 vs 通用多语言模型"的差异
  ✓ Whisper输出繁体，已通过zhconv繁简转换归一化
```

## 第10页：流式ASR

**排版**：左侧表格 + 右侧RTF柱状图

**标题栏**：`实验结果⑤：流式ASR — StreamingASR模块评估`

**左侧结果表格**：
```
┌──────────────┬──────────┬──────────┐
│ 分块大小      │ CER(%)   │ RTF      │
├──────────────┼──────────┼──────────┤
│ 200ms        │ 4.71%    │ 0.0712   │
│ 500ms        │ 5.93%    │ 0.0745   │
│ 1000ms       │ 6.93%    │ 0.1077   │
└──────────────┴──────────┴──────────┘
(数据来自THCHS-30测试集500条)
```

**右侧分析要点**：
```
▪ 200ms块CER=4.71%，最接近全音频基线(4.06%)
  静音阈值=1200ms，需6个连续静音200ms块才flush
  连续语音中几乎从未触发flush
  测试脚本自动降级成全音频识别（兜底机制）→ 低CER

▪ 500/1000ms块CER偏高 ≠ 大块本身不好
  静音阈值按"块数"计算，大块对应更少的计数
  1000ms块：1200/1000 ≈ 1 → 1个低能量块就触发flush
  THCHS-30语音中自然的能量波动（辅音段）也会被误判为静音
  → 提前刷出缓冲区 → 识别片段变短 → CER升高

▪ 这是能量检测VAD的局限性，不是大块的问题
  200ms块有更多"投票窗口"，抵御能量波动能力更强
  如果用FSMN-VAD模型替代能量检测，大块表现会改善
```

## 第11页：错误分析

**排版**：上方错误分布饼图，下方3个典型案例

**标题栏**：`实验结果分析：错误案例与模型局限性`

**错误分布**（基于THCHS-30高CER样本）：
```
▪ 高错误样本特征：
  - 含生僻字的音频（古诗词、文言文）
    例："选一本好书使你罹小恙而顿愈" CER=19.35%
  - 语速过快或发音含糊的样本
  - 短音频（<3秒，上下文不足）

▪ 典型错误类型：
  同音字替换为主（藏→长、圏→内）
```

**典型案例**（建议放3个对比框）：
```
案例1：生僻字
参考：王英汉被枪毙后部分余孽深藏起来
识别：王英汉被枪毙后部分余孽深长起来
错误类型：同音字替换（"藏"→"长"）

案例2：粤语口音（20段真实语音）
参考：[粤语普通话对话]
识别：完全为空/乱码
原因：Paraformer训练数据以标准普通话为主
     粤语zh/ch/sh→z/c/s、n/l混淆、声调差异

案例3：古诗词
参考：选一本好书使你罹小恙而顿愈
识别：选一本好书，使你立小样而遁
错误类型：多字+同音替换
```

**底部总结**：
```
▪ 系统能做什么：标准普通话朗读语音，CER可控在5%以内
▪ 系统不能做什么：非标准口音、高噪Babble场景
▪ 如何改进：口音数据微调、N-gram语言模型
```
